# v1.13.0 Migration Guide

In addition to the [standard migration procedure](../migration.md), perform the following steps.

## cjobctl CLI Command Changes

`cjobctl counters list` has been migrated to `cjobctl jobs counters`. If existing scripts or procedures use the old command, update them accordingly.

## Remove `cohortName` / `lendingLimit` from ClusterQueue

The ClusterQueue design has been revised: `cohortName: cjob-cohort` and the `lendingLimit: "0"` previously set on GPU flavors have been removed. Because the Dispatcher always sets a per-flavor `nodeSelector` on Job Pods, Kueue's flavor matching structurally prevents any job from consuming another flavor's quota, and these settings were effectively no-ops.

In existing environments, update the ClusterQueue resource with the following steps:

```bash
kubectl edit clusterqueue cjob-cluster-queue
```

Apply these changes:

1. Remove the `spec.cohortName` line
2. Remove all `spec.resourceGroups[0].flavors[*].resources[*].lendingLimit` lines

After the change, confirm that the `(lendingLimit: 0)` annotations disappear from the output of `cjobctl cluster show-quota`. Kueue's admission behavior does not change, so there is no impact on running or pending jobs.

## Additional Steps When Overriding the Dispatcher Metrics Port

A new `DISPATCHER_METRICS_PORT` key has been added to base (default: `9090`). The default matches the Dispatcher Deployment's `containerPort: 9090` and the Pod annotation `prometheus.io/port: "9090"` in base, so no action is required if you use the default port.

To change the port, editing only the ConfigMap key causes a mismatch with the Pod's `containerPort` / annotation, and Prometheus will fail to scrape. The following three places must be overridden together in your overlay:

1. **ConfigMap**: Add `DISPATCHER_METRICS_PORT: "<port>"` to your overlay's `configmap-cjob-config.yaml`
2. **Deployment containerPort**: Add the following patch to your overlay's `kustomization.yaml`
   ```yaml
   - patch: |
       - op: replace
         path: /spec/template/spec/containers/0/ports/0/containerPort
         value: <port>
     target:
       kind: Deployment
       name: dispatcher
   ```
3. **Pod annotation** (only if you use annotation-based Prometheus discovery): override it with the same patch mechanism
   ```yaml
   - patch: |
       - op: replace
         path: /spec/template/metadata/annotations/prometheus.io~1port
         value: "<port>"
     target:
       kind: Deployment
       name: dispatcher
   ```

Environments using Prometheus Operator (`base/prometheus-operator/podmonitor-dispatcher.yaml`) reference the port by name (`metrics`), so no change is required on the PodMonitor side even if the port number is changed.

After applying, restart the Dispatcher with `kubectl rollout restart deployment/dispatcher -n cjob-system`, and verify scraping via `curl http://<pod-ip>:<port>/metrics` or the Prometheus Targets page.
