# v1.13.0 移行手順

[標準移行手順](../migration.md) に加えて以下の手順を実施する。

## cjobctl CLI コマンド変更

`cjobctl counters list` は `cjobctl jobs counters` に移行された。既存のスクリプトや手順書で旧コマンドを使用している場合は更新すること。

## ClusterQueue から `cohortName` / `lendingLimit` を削除する

ClusterQueue の設計を見直し、`cohortName: cjob-cohort` と GPU flavor に設定していた `lendingLimit: "0"` を削除した。Dispatcher が Job Pod に flavor ごとの `nodeSelector` を必ず設定しているため、Kueue の flavor 照合によって別 flavor の quota を消費することは構造的に発生せず、これらの設定は実質的に no-op だった。

既存環境では ClusterQueue リソースを以下の手順で更新する:

```bash
kubectl edit clusterqueue cjob-cluster-queue
```

以下の変更を行う:

1. `spec.cohortName` の行を削除する
2. `spec.resourceGroups[0].flavors[*].resources[*].lendingLimit` の行をすべて削除する

変更後、`cjobctl cluster show-quota` の出力から `(lendingLimit: 0)` の併記が消えることを確認する。Kueue 側の admission 挙動は変化しないため、実行中・pending のジョブへの影響はない。

## Dispatcher メトリクスポートを上書きする場合の追加手順

base に `DISPATCHER_METRICS_PORT` キーが追加された（デフォルト値: `9090`）。デフォルト値は base の Dispatcher Deployment の `containerPort: 9090` および Pod アノテーション `prometheus.io/port: "9090"` と一致しているため、デフォルトポートを使う場合は対応不要。

別ポートに変更したい環境では、ConfigMap のキーだけを書き換えると Pod の `containerPort` / annotation とズレて Prometheus から scrape できなくなる。以下 3 箇所を overlay で同時に上書きする必要がある:

1. **ConfigMap**: overlay の `configmap-cjob-config.yaml` に `DISPATCHER_METRICS_PORT: "<port>"` を追加する
2. **Deployment の containerPort**: overlay の `kustomization.yaml` に次の patch を追加する
   ```yaml
   - patch: |
       - op: replace
         path: /spec/template/spec/containers/0/ports/0/containerPort
         value: <port>
     target:
       kind: Deployment
       name: dispatcher
   ```
3. **Pod annotation**（annotation-based Prometheus discovery を使っている場合）: 同様に patch で上書きする
   ```yaml
   - patch: |
       - op: replace
         path: /spec/template/metadata/annotations/prometheus.io~1port
         value: "<port>"
     target:
       kind: Deployment
       name: dispatcher
   ```

Prometheus Operator を使っている環境（`base/prometheus-operator/podmonitor-dispatcher.yaml`）はポート名 `metrics` で参照しているため、ポート番号を変更しても PodMonitor 側の変更は不要。

適用後、`kubectl rollout restart deployment/dispatcher -n cjob-system` で再起動し、`curl http://<pod-ip>:<port>/metrics` または Prometheus の Targets 画面で scrape できていることを確認する。
