# ユーザーガイド

## 1. ジョブ数の制限

CJob では、システムの安定性と公平性を保つために、ユーザーごとにジョブ数の上限が設けられています。

### 1.1 投入できるジョブ数の上限（500 件）

`cjob add` や `cjob sweep` で投入できるジョブの数には上限があります。まだ完了していないジョブ（待機中・実行中・キャンセル処理中のもの）の合計が **500 件** を超えると、新しいジョブを投入できなくなります。

上限に達した場合は、実行中のジョブが完了するのを待つか、不要なジョブを `cjob cancel` でキャンセルしてください。

### 1.2 同時に実行できるジョブ数の上限（32 件）

一度に実行状態になれるジョブは **最大 32 件** です。33 件目以降のジョブは待機状態（QUEUED）のまま順番を待ち、実行中のジョブが完了すると自動的に次のジョブが開始されます。

基本的には投入した順番に実行されますが、リソースの空き状況によっては後から投入した小さいジョブが先に実行される場合があります。

### 1.3 完了済みジョブの保持上限（100 件）

実行が終わったジョブ（成功・失敗）は、しばらくの間（最大 3 時間）システム内部に残ります。実行中のジョブと合わせて **100 件** を超えると、新しいジョブの実行が一時的に遅れる場合があります。

通常は自動的に解消されますが、大量のジョブを連続して実行した場合は、不要な完了済みジョブを `cjob delete` で削除すると、新しいジョブの実行がスムーズになります。

```bash
# 完了済みジョブをまとめて削除
cjob delete --all
```

### 1.4 制限のまとめ

| 制限の内容 | 上限 | 上限に達したときの動作 |
|---|---|---|
| 投入できるジョブ数 | 500 件 | 新しいジョブを投入できなくなる |
| 同時に実行できるジョブ数 | 32 件 | 超過分は待機状態で順番を待つ |
| 完了済みジョブの保持数 | 100 件 | 新しいジョブの実行開始が遅れる場合がある |

## 2. sweep を使った効率的なジョブ実行

パラメータを変えながら同じプログラムを何度も実行したい場合は、`cjob sweep` を使うと便利です。

### 2.1 基本的な使い方

たとえば、パラメータ `--trial` の値を 0 から 99 まで変えて 100 回実行したい場合を考えます。

`cjob add` で 1 件ずつ投入すると 100 回コマンドを実行する必要がありますが、`cjob sweep` なら 1 回のコマンドで済みます。

```bash
# 100 回分のジョブを一度に投入し、10 件ずつ並列で実行する
cjob sweep -n 100 --parallel 10 -- python main.py --trial _INDEX_
```

- `-n 100`: 全部で 100 回実行する
- `--parallel 10`: 同時に 10 件まで並列で実行する
- `_INDEX_`: 各実行に自動で 0, 1, 2, ... , 99 の番号が割り当てられる

### 2.2 `_INDEX_` の活用方法

`_INDEX_` は実行ごとに異なる番号（0 から始まる連番）に自動で置き換えられます。この番号を使ってパラメータを変えたり、入力ファイルを切り替えたりできます。

```bash
# 番号をそのままパラメータとして使う
cjob sweep -n 50 --parallel 5 -- python train.py --seed _INDEX_

# 番号で入力ファイルを切り替える（run.sh の中で番号に応じたファイルを読み込む）
cjob sweep -n 10 --parallel 5 -- bash run.sh _INDEX_
```

上の例では `run.sh` に番号が引数として渡されます。スクリプトの中でその番号を使って入力ファイルを切り替えることができます。

```bash
# run.sh の中身の例（$1 に _INDEX_ の値が入る）
python train.py --config configs/config_${1}.yaml
```

この場合、番号 0 のジョブは `config_0.yaml`、番号 1 のジョブは `config_1.yaml` を読み込みます。

スクリプトファイルの中では、引数の代わりに `$CJOB_INDEX` という変数名で番号を直接参照することもできます。

```bash
# run.sh の中身の例（$CJOB_INDEX を使う場合）
python train.py --config configs/config_${CJOB_INDEX}.yaml
```

### 2.3 並列数の選び方

`--parallel` の値は、同時に実行できるジョブ数の上限（32 件）以下に設定してください。他のジョブも同時に実行したい場合は、その分を差し引いた値にするとよいでしょう。

```bash
# 他にジョブがなければ最大 32 まで並列にできる
cjob sweep -n 100 --parallel 32 -- python main.py --trial _INDEX_

# 他のジョブ用に枠を残しておく場合
cjob sweep -n 100 --parallel 20 -- python main.py --trial _INDEX_
```

### 2.4 sweep のログ確認

sweep で実行した各ジョブのログは、番号を指定して個別に確認できます。

```bash
# 全タスクのログをまとめて表示
cjob logs 3

# 番号 5 のタスクのログだけ表示
cjob logs 3 --index 5

# 番号 5 のタスクのログをリアルタイムで追跡
cjob logs --follow 3 --index 5
```

### 2.5 sweep のキャンセル

sweep ジョブをキャンセルすると、進行中の全タスクがまとめて中止されます。特定の番号だけを中止することはできません。

```bash
cjob cancel 3
```
