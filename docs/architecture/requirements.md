# 機能要件

## 1. 提供したいジョブキューシステムの機能

### 1.1 ユーザー向け基本機能

本システムが提供する主要機能は次の通りである。

- シェルコマンドをジョブとして投入する
- パラメータスイープをジョブとして投入する（Indexed Job による並列実行）
- 投入済みジョブの一覧を確認する
- 個別ジョブの状態を確認する
- ジョブをキャンセルする
- ジョブの実行を保留する・保留を解除する（個別・一括・全件）
- 完了済みジョブを削除する（個別・一括）
- ジョブのログを確認する（リアルタイム追跡含む）
- 自身のリソース使用状況を確認する（日別の CPU・メモリ・GPU 消費量）
- 全ジョブ履歴・ログを削除し job_id をリセットする
- CLI バイナリをセルフアップデートする
- 将来的に workflow engine から API 経由で利用できる

### 1.2 ジョブ投入時に再現したい情報

ユーザーがジョブを投入した時点の以下の情報をジョブ実行時に再現する。

- 作業ディレクトリ (`cwd`)
- export 済み環境変数（仮想環境の `PATH` / `VIRTUAL_ENV` を含む。ユーザー設定で除外指定した変数を除く）
- 実行コマンド文字列

### 1.3 実行制御上の機能

- ジョブの一時保管
- ジョブの Kubernetes Job への変換
- Kueue による admission 制御
- namespace ごとの dispatch 数制御
- namespace ごとの ResourceQuota による意図しない無制限消費の防止（安全網）
- 実行状態の追跡
- Kubernetes Job / Pod 状態と内部状態 DB の整合
